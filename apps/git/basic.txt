GIT 的几个基本概念
1.  GIT 目录
	  一般指项目根目录下的.git 目录
	  用于保存所有元配置和历史信息

2. 工作目录
	 指项目根目录, 包括 git 目录和项目代码目录
	 用于存储你当前分支内容
	 当在不同分支之间切换时，工作目录内容会被不断删除和替换
	 而所有分支内容及历史信息都保存在GIT 目录下

3. GIT 索引
     是一个在工作目录和GIT 仓库之间的暂存区。
     如果你创建了一个commit, 提交的是GIT索引中的内容，而不是工作目录的内容
     相关命令:
      1> git status
      2> git add
      3> git commit
      
4. 有两种方法可以得到一个本地git 库
	 1> 从本地目录创建一个git 库
	 	git init
	 	
	 2> 从远端克隆一个git 库
	 	git clone git@github.com:michaelliao/learngit.git
